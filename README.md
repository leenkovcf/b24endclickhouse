# b24endclickhouse

Локальное веб-приложение для выгрузки данных из Bitrix24 (через BI Connector) в ClickHouse. Подходит для последующей аналитики в Yandex DataLens или других BI-системах.

## Возможности

- Выгрузка любых CRM-сущностей: сделки, лиды, контакты, компании, задачи, звонки, смарт-процессы и т.д.
- Фильтрация по воронкам, стадиям и отдельным полям
- Пошаговый прогресс выгрузки с отображением строк по каждому дню
- Автоматическое создание таблиц в ClickHouse при первой выгрузке
- Выгрузка по расписанию (раз в час / сутки / неделю)
- Автозагрузка смарт-процессов через REST API Webhook
- Сохранение настроек и выбора фильтров между сессиями

---

## Требования

- Python 3.10+
- Доступ к Bitrix24 с включённым BI Connector
- Доступ к ClickHouse (Yandex Cloud или self-hosted)

## Установка

### 1. Клонировать репозиторий

```bash
git clone https://github.com/leenkovcf/b24endclickhouse.git
cd clickhouse
```

### 2. Установить зависимости

**Windows (автоматически):**

```
install.bat
```

**Вручную:**

```bash
python -m venv venv
venv\Scripts\activate      # Windows
# source venv/bin/activate  # Linux / macOS
pip install -r requirements.txt
```

### 3. Запустить приложение

**Windows (автоматически):**

```
start.bat
```

**Вручную:**

```bash
venv\Scripts\activate
uvicorn main:app --host 127.0.0.1 --port 8000
```

Приложение откроется по адресу: **http://localhost:8000**

## Настройка

### Bitrix24

1. Перейдите в CRM → BI-аналитика → Управление ключами
2. Создайте новый BI-ключ или скопируйте существующий
3. Адрес портала — например `example.bitrix24.ru` (без https://)

**REST API Webhook** (необязательно):
Нужен только для автозагрузки смарт-процессов и фильтрации по воронкам/стадиям.
Создаётся в: Профиль → Разработчикам → Входящий вебхук (права: CRM).

### ClickHouse

Поддерживаются:
- Yandex Cloud Managed ClickHouse (порт 8443, SSL)
- Self-hosted ClickHouse (порт 8123 или 9000)

Заполните хост, порт, базу данных, пользователя и пароль в разделе **Подключение**.

## Использование

### Разовая выгрузка

1. Откройте раздел **Подключение** → заполните данные → нажмите **Проверить подключение**
2. Перейдите в **Отправка данных**
3. Выберите сущность (например, «Сделки»)
4. Выберите тип даты и период
5. При необходимости настройте фильтры (воронки, стадии, поля)
6. Нажмите **Запустить выгрузку**

Прогресс отображается в реальном времени — по каждому дню показывается количество загруженных строк.

### Автоматическая выгрузка по расписанию

1. Перейдите в раздел **Расписание**
2. Выберите периодичность: раз в час / сутки / неделю
3. Выберите сущность, тип даты и глубину (сколько дней назад выгружать)
4. Нажмите **Сохранить расписание**

Расписание работает пока приложение запущено.

### Смарт-процессы

В разделе **Подключение → Смарт-процессы и кастомные сущности**:
- **Загрузить автоматически** — загружает все смарт-процессы с портала через REST Webhook
- **Добавить вручную** — введите код таблицы (`crm_dynamic_items_105`) и название

ID смарт-процесса виден в URL при открытии его настроек в Bitrix24.

## Таблицы в ClickHouse

Таблицы создаются автоматически при первой выгрузке с именем сущности (например, `crm_deal`).
Типы столбцов определяются автоматически: `Int64`, `Float64`, `Date`, `DateTime`, `String`.

При повторной выгрузке данные **дописываются** (APPEND), а не перезаписываются — используйте фильтр по дате на стороне ClickHouse/DataLens при необходимости дедупликации.
